# 大明浮生志3 游戏设计文档

版本：0.1.0
目标平台：Android Native / 后续可扩展到 TapTap Code 版本
当前阶段：立项框架 + MVP 骨架
设计原则：与《大明浮生志2》保持精神传承，但美术、音效、主循环和玩法结构必须明显不同。

---

## 1. 项目定位

### 1.1 一句话定位

《大明浮生志3》是一款以明末乱世为背景的县域宗族沙盘游戏。玩家带领一个宗族，在县域地图上经营田庄、周旋官府、安抚乡民、抵御流寇、培养子弟，并在王朝崩塌前选择家族命运。

### 1.2 与二代的核心区别

| 维度 | 大明浮生志2 | 大明浮生志3 |
|---|---|---|
| 游戏核心 | 宗族经营 + 族谱传承 | 县域沙盘 + 宗族政治 + 乱世抉择 |
| 主页面 | 族谱 / 宗族 / 族人 / 经营 | 县域地图作为默认核心场景 |
| 资源循环 | 月结资源增长与消耗 | 地点控制、人物派遣、势力关系、危机扩散 |
| 人物价值 | 属性与状态管理 | 性格、房支、任务、关系、抉择后果 |
| 美术气质 | 浅色古籍、水墨族谱 | 深色案牍、朱砂批注、旧地图、烛火阴影 |
| 音频气质 | 古风背景乐 + 基础 UI 音效 | 动态局势氛围 + 县域环境音 + 官衙/军报音效 |
| 目标 | 宗族延续与品级成长 | 多路线结局：士族、商族、堡主、勤王、海外、割据 |

### 1.3 设计关键词

- 明末县域
- 宗族沙盘
- 家族政治
- 乱世地方治理
- 人物派遣事件
- 县衙案牍
- 朱砂批注
- 多路线结局

## 2. 体验目标

### 2.1 玩家每 3 分钟应体验到

1. 县域局势发生变化。
2. 至少一个族人被安排或卷入事件。
3. 一个地点状态、势力关系或资源指标发生可感知变化。
4. 玩家做一次有代价的选择。

### 2.2 玩家每 15 分钟应体验到

1. 一个年度目标或阶段目标完成/失败。
2. 家族内部出现矛盾或成长。
3. 县域中出现新的危机点或机会点。
4. 路线倾向发生改变，例如更偏商族、士族、堡主或勤王。

### 2.3 玩家一局的长期体验

玩家应感到自己不是在单纯刷资源，而是在写一部家族史：

- 哪些族人被培养成栋梁？
- 哪些房支兴起或衰落？
- 哪次灾荒改变了家族路线？
- 是选择忠于朝廷、保境安民、富甲一方，还是远渡海外？

## 3. 世界观与背景

### 3.1 时间框架

默认起点：洪武末年至明中后期之间可选。
核心压力期：万历后期、天启、崇祯、甲申。
终局判断：明亡前后、南明、清初、海外迁徙或地方割据。

### 3.2 空间框架

核心空间不再是抽象宗族页，而是县域地图：

- 自家祠堂

- 寨堡
- 山道
- 码头
- 流寇营地
- 官军驻点
- 商路节点

县域之外通过事件和远行系统表现：

## 4. 美术设计

### 4.1 总体方向

三代不继续二代的浅色纸张水墨主视觉，而改为：

- 深棕、墨黑、旧绢布底
- 朱砂红作为重点色
- 烛火金用于高亮
- 青灰用于危机和阴影
- 纸张磨损、案牍纹理、旧地图边缘

视觉目标：像一套明末县衙案卷、宗族密札和战时军报混合在一起。

### 4.2 UI 风格

#### 一级 UI

- 县域地图：旧地图底纹 + 可点击地点图标。
- 顶栏：年份、月份、天候、局势等级。
- 底栏：县域、宗族、人物、政略。
- 右侧：急报、任务、危机、路线倾向。

#### 二级 UI

- 案牍卡片：深色边框、旧纸底、朱砂批注。
- 军报弹窗：红黑色调、急报印章。
- 家族事件：族谱纸、手写批注。
- 官府事件：县衙牌匾、印章、文书。

### 4.3 关键页面美术

| 页面 | 美术方向 |
|---|---|
| 主菜单 | 夜色县城、远处火光、宗祠灯笼、旧地图叠影 |
| 开局创建 | 族谱残卷 + 县域地图局部 + 朱砂选择印记 |
| 县域 | 卷轴地图，可见村、田、衙、寨、集、山道 |
| 宗族 | 深色宗祠、牌位、房支树 |
| 人物 | 明代人物卡，身份服饰和年龄阶段变化 |
| 产业 | 地点图标 + 地块状态，而不是纯列表 |
| 政略 | 官府、士绅、乡民、流寇、商帮关系盘 |
| 天下 | 年表、战报、地图态势、路线终局 |

## 5. 音效与音乐设计

### 5.1 BGM 系统

三代使用局势驱动音乐，不再只是菜单/和平/危机/战斗切换。

| 局势 | 音乐方向 |
|---|---|
| 太平经营 | 古琴、洞箫、轻鼓 |
| 县域繁荣 | 琵琶、笛、轻市井氛围 |
| 灾荒 | 低弦、风声、远雷 |
| 官府压迫 | 低鼓、木鱼、官衙堂鼓 |
| 流寇逼近 | 急鼓、低号角、弦乐持续音 |
| 战斗 | 战鼓、金属碰撞、短促号角 |
| 葬礼 | 埙、低笛、风声、远钟 |
| 终局 | 钟鼓、合唱或长笛，根据结局路线变化 |

### 5.2 环境音

- 县域地图：风声、鸟鸣、远处人声。
- 集市：叫卖、铜钱、脚步。
- 县衙：惊堂木、翻案卷、盖官印。
- 宗祠：钟磬、烛火、木门。
- 流寇危机：马蹄、火声、远鼓。
- 灾荒：风沙、咳嗽、低语。

### 5.3 UI 音效

| 操作 | 音效 |
|---|---|
| 页面切换 | 翻册声 |
| 确认 | 朱印盖章 |
| 警告 | 惊堂木 / 木梆 |
| 建设 | 锤木声 |
| 任命 | 展开文书 |
| 战报 | 急鼓 + 纸卷展开 |
| 资源变动 | 铜钱、粮袋、布卷 |
| 人物死亡 | 低钟 |

## 6. 核心玩法结构

### 6.1 四层循环

#### 第一层：宗族内政

- 族谱
- 房支
- 族长继承
- 家训
- 族规
- 婚姻
- 生育
- 教育
- 家族矛盾

新增重点：房支系统。

房支类型：

- 主房
- 二房
- 庶支
- 商支
- 武支
- 书香支

每个房支拥有：

- 人口
- 威望
- 财富倾向
- 路线偏好
- 对族长忠诚

#### 第二层：县域经营

县域地图上有地点，每个地点有等级、风险、控制度和任务槽。

地点示例：

| 地点 | 作用 |
|---|---|
| 祠堂 | 宗族凝聚、威望、族规 |
| 田庄 | 粮食、劳力、灾害风险 |
| 集市 | 银两、商品、商路 |
| 县衙 | 官府关系、税赋、徭役 |
| 书院 | 读书、科举、人脉 |
| 医馆 | 疾病、寿命、瘟疫 |
| 寨堡 | 防御、乡勇、流寇 |
| 码头 | 海贸、迁徙、外部路线 |
| 山道 | 流寇、采药、探险 |

#### 第三层：人物派遣

每月玩家安排关键族人执行任务。

任务拥有：

- 负责人
- 地点
- 时间
- 成功率
- 风险
- 性格修正
- 房支影响
- 结果事件

任务示例：

- 管理田庄
- 护送商队
- 拜访县令
- 赈济灾民
- 修筑寨堡
- 招募乡勇
- 刺探流寇
- 联姻谈判
- 读书备考
- 求医问药

#### 第四层：天下大势

大势由时间与玩家选择共同推动。

指标：

- 朝廷稳定
- 流寇压力
- 赋税压力
- 灾荒程度
- 清军压力
- 海贸机会

这些指标影响县域事件池和结局路线。

## 7. 核心系统

### 7.1 县域地图系统

#### 地点数据

每个地点包含：

- id
- 名称
- 类型
- 等级
- 控制度
- 风险值
- 产出
- 可用任务
- 当前危机

#### 地点状态

- 平稳
- 繁荣
- 荒废
- 被盗匪威胁
- 被官府征用
- 疫病蔓延
- 商路阻断
- 战火波及

### 7.2 族人性格系统

性格影响任务结果、事件选择和家族矛盾。

性格列表：

- 忠厚
- 贪财
- 好学
- 尚武
- 怯懦
- 野心
- 仁善
- 狡黠
- 刚烈
- 圆滑

示例影响：

- 贪财：经商收益提高，但可能私吞。
- 尚武：战斗任务成功率提高，但冲突事件概率提高。
- 仁善：赈济效果提高，但资源消耗增加。
- 野心：仕途成长快，但房支争权概率提高。

### 7.3 家族矛盾系统

指标：

- 族长威望
- 房支忠诚
- 族产分配满意度
- 嫡庶矛盾
- 婚姻不满
- 继承争端

事件：

- 分家争产
- 族老逼宫
- 庶支出走
- 商支私藏银两
- 武支要求扩军
- 书香支要求建书院

### 7.4 地方关系系统

势力：

| 势力 | 影响 |
|---|---|
| 官府 | 税收、徭役、科举、官职 |
| 士绅 | 声望、联姻、土地、人脉 |
| 乡民 | 劳力、民心、治安、流民 |
| 流寇 | 劫掠、战斗、招安、威胁 |
| 商帮 | 商路、银两、货物、海外路线 |
| 军镇 | 防御、从军、兵器、战报 |

关系值范围：-100 到 100。

### 7.5 事件叙事系统

事件结构：

- 标题
- 背景文本
- 触发条件
- 参与人物
- 选择项
- 即时效果
- 延迟后果
- 路线倾向变化

示例事件：

> 县令急召，言流寇将至，命各族出粮三百石。族老劝你推脱，商支主张花钱买通，武支愿领乡勇守寨。

选择：

1. 出粮助官：官府关系上升，粮食下降。
2. 暗中拖延：风险保留，族内忠诚下降。
3. 买通县吏：银两下降，官府关系小幅上升。
4. 组织乡勇自保：武支威望上升，流寇敌意上升。
5. 联合其他宗族谈判：士绅关系上升，成功率受声望影响。

### 7.6 多路线结局

| 路线 | 触发倾向 | 结局方向 |
|---|---|---|
| 耕读传家 | 科举、书院、士绅 | 名门望族 / 宰辅世家 |
| 富甲江南 | 商路、码头、钱庄 | 皇商巨族 / 海贸世家 |
| 聚族自保 | 寨堡、乡勇、屯粮 | 乱世堡主 / 保境安民 |
| 从龙勤王 | 官府、军功、抗寇 | 忠烈之家 / 中兴勋族 |
| 割据一方 | 军备、流寇、县域控制 | 地方诸侯 / 乱世军阀 |
| 远渡海外 | 码头、船队、海贸 | 南洋宗族 / 海外新家园 |
| 隐世避祸 | 低声望、低风险、避战 | 山林遗族 / 香火延续 |

## 10. 代码架构要求

### 10.1 模块化原则

三代代码必须独立于二代核心逻辑，避免继续把所有内容堆进现有文件。

建议包结构：

```text
com.daming.fushengzhi2.v3
├── data
│   ├── V3Models.kt
│   └── V3Content.kt
├── logic
│   ├── V3GameEngine.kt
│   └── V3GameController.kt
└── ui
    ├── V3GameScreen.kt
    ├── V3CreateScreen.kt
    ├── V3CountyScreen.kt
    ├── V3ClanScreen.kt
    ├── V3PeopleScreen.kt
    └── V3StrategyScreen.kt
```

### 10.2 文件大小限制

- 单文件超过 1000 行必须拆分。
- 页面和逻辑必须分离。
- 数据定义和内容表必须分离。
- 事件系统后续单独拆成 `events` 包。

### 10.3 兼容策略

- 二代代码保留可运行。
- 主菜单新增“三代框架”入口。
- 三代使用独立控制器和状态，不破坏二代存档。
- 后续再决定是否迁移包名、应用名和图标。

## 12. 当前执行标准

1. 不做简单换皮。
2. 不把二代页面复制后改标题当三代。
3. 新系统必须围绕县域地图、人物派遣、势力关系和多结局路线设计。
4. 所有新增代码优先模块化。
5. 第一版允许内容未满，但框架必须能支撑完整游戏。
6. 每轮改动后必须检查构建风险；未确认前不提交、不推送。

## 13. 可上架验收标准

### 13.1 内容量

- 县域地点不少于 9 个，且每个地点至少有派遣、营建、风险三层玩法。
- 族人不少于 10 名，具备属性、性格、功绩、疲劳和派遣成长。
- 房支不少于 6 个，具备忠诚、财富、势力、怨气和议事事件。
- 事件至少覆盖：县衙催粮、流寇、流民、瘟疫、书院、海商、商帮、房支争产、房支逼议、县中流言。
- 年度目标、路线倾向、结局预判必须能持续驱动玩家决策。
- 正式终局必须能给出路线、等级、分数、叙事文本和可重新开局流程。

### 13.2 表现质量

- 三代视觉必须与二代明显区分：深色案牍、朱砂批注、旧地图、烛火氛围。
- 主菜单入口、创建页、县域页、宗族页、人物页、政略页必须信息清晰、按钮可读、后果可理解。
- 事件选择必须展示关键后果，避免玩家盲选。
- 音效至少覆盖点击、切页、月结、事件选择，后续发布前替换为三代专属音效。

### 13.3 稳定性

- 三代使用独立存档，不破坏二代存档。
- 所有新增数据类必须兼容序列化。
- 本地静态检查必须通过。
